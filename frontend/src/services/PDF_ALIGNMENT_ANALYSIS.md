# PDF 정렬 문제 분석 및 해결

## 문제 원인

### 1. 화면과 PDF의 단위 불일치

**화면 (StoryBoardBody.tsx):**
- 일반 컷 번호 영역: `w-16 md:w-20` 
  - 작은 화면: `w-16` = 64px (4rem) ≈ 16.93mm (96 DPI 기준)
  - 큰 화면: `w-20` = 80px (5rem) ≈ 21.17mm (96 DPI 기준)
- 이미지만 추가 그룹: `pl-16 md:pl-20`
  - 작은 화면: `pl-16` = 64px ≈ 16.93mm
  - 큰 화면: `pl-20` = 80px ≈ 21.17mm

**PDF (이전):**
- 컷 번호 영역: `width: 15mm`
- 이미지만 추가 그룹: `padding-left: 15mm`

**문제점:**
- 화면은 64px (≈16.93mm) 또는 80px (≈21.17mm) 사용
- PDF는 15mm 사용 → 약 2-6mm 차이 발생

### 2. Gap 간격 차이

**화면:**
- 일반 컷: `gap-3 md:gap-4` (12px/16px) = 3mm/4mm
- 이미지만 추가: `gap-3 md:gap-4` (12px/16px) = 3mm/4mm

**PDF:**
- 일반 컷: `gap: 3mm` (고정)
- 이미지만 추가: `gap: 2.5mm` (다름!)

### 3. 컷 번호가 없는 항목 처리

**일반 항목 (이미지 추가):**
- 컷 번호 영역은 항상 존재 (`w-16 md:w-20`)
- 컷 번호가 없으면 투명 텍스트로 공간 유지
- `gap-3 md:gap-4`로 컷 번호 영역과 이미지 사이 간격

**이미지만 추가:**
- 컷 번호 영역이 없음
- `pl-16 md:pl-20`로 왼쪽 여백만 적용
- `gap-3 md:gap-4`는 이미지 간 간격

## 해결 방법

### 적용된 수정

1. **컷 번호 영역 너비 조정:**
   - `width: 15mm` → `width: 17mm` (64px ≈ 16.93mm에 근사)

2. **이미지만 추가 그룹 패딩 조정:**
   - `padding-left: 15mm` → `padding-left: 17mm`

3. **일관된 gap 사용:**
   - 이미지만 추가 그룹도 `gap: 3mm`로 통일 (현재 2.5mm)

### 추가 권장 사항

만약 여전히 정렬이 맞지 않는다면:

1. **더 정확한 값 사용:**
   - 화면의 `w-16` (64px)를 정확히 계산하면 16.93mm
   - 화면의 `w-20` (80px)를 정확히 계산하면 21.17mm
   - PDF는 고해상도로 인쇄되므로, 평균값인 19mm 사용 고려

2. **gap 일치:**
   - 이미지만 추가 그룹의 `gap: 2.5mm` → `gap: 3mm`로 변경

3. **실제 렌더링 측정:**
   - 화면에서 실제 렌더링된 컷 번호 영역의 너비를 브라우저 개발자 도구로 측정
   - 측정된 값을 PDF에 정확히 반영
